# Firebase 보안 규칙 적용 가이드 (`srk-mt-default-rtdb`)

Firebase가 보내는 **"안전하지 않은 규칙" 경고 메일**을 멈추고, DB URL을 아는 외부인이
크루 데이터를 마음대로 읽거나 통째로 지우지 못하게 막기 위한 설정입니다.

## 규칙이 하는 일 (`database.rules.json`)

```json
{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null && (!data.exists() || newData.hasChildren(['members']))"
  }
}
```

- **`.read: auth != null`** — 로그인 토큰이 있어야만 읽기 가능 → 떠돌이 봇·스캐너 차단, 경고 메일 멈춤.
- **`.write`** — 로그인 토큰이 있고, **쓰기 후에도 `members`(크루 명단)가 남아 있을 때만** 허용.
  - 일반 쓰기(공지·일정·당구기록·정산 등)는 명단을 건드리지 않으니 전부 통과.
  - `최초 시드(빈 DB)`도 통과 (`!data.exists()`).
  - **루트(`/`) 통째 삭제, 명단(`members`) 통째 삭제는 거부** → 한 번에 전체 데이터가 날아가는 사고/반달리즘 방지.

> ⚠️ 부수효과: 명단을 비우는(마지막 1명까지 삭제) 동작은 막힙니다. 실사용 플로우엔 없으니 문제 없음.

이 앱은 Firebase 인증을 쓰지 않고 **익명 로그인(anonymous)** 으로 토큰만 발급받습니다.
PIN은 여전히 앱 단에서만 검증하므로, "앱을 연 사람"까지 막지는 못합니다(공격면을
`인터넷 전체 → 앱 접속자`로 좁히는 게 목적). 더 강한 보안이 필요하면 정식 인증 도입이 필요합니다.

## 적용 순서 (중요 — 이 순서를 지켜야 라이브 앱이 안 끊깁니다)

1. **Firebase 콘솔 → Authentication → Sign-in method → 익명(Anonymous) → 사용 설정.**
   (이걸 안 켜면 4단계 후 앱이 로그인 토큰을 못 받아 전체가 멈춥니다.)
2. 이 PR의 코드(`index.html` + `app.js`, 익명 로그인 추가)를 **먼저 배포**한다.
   - 이 시점엔 규칙이 아직 열려 있어 앱은 평소대로 동작하고, 익명 로그인만 조용히 수행됩니다.
3. 앱을 열어 정상 동작 확인(데이터 로딩·입력 OK, 콘솔에 `익명 인증 실패` 경고 없음).
4. **그다음** Firebase 콘솔 → Realtime Database → **규칙(Rules)** 탭에 `database.rules.json`
   내용을 붙여넣고 **게시(Publish)**. (또는 `firebase deploy --only database`)
5. 다시 앱을 열어 읽기/쓰기 정상 확인. 외부에서 토큰 없이 접근하면 차단됩니다.
6. 경고 메일이 멈춥니다(반영까지 약간의 시간차 있을 수 있음).

> ❗ **순서 주의:** 4번(규칙 게시)을 2번(코드 배포)보다 **먼저** 하면, 아직 익명 로그인을
> 안 하는 라이브 앱이 권한오류로 멈춥니다. 반드시 코드 배포 → 확인 → 규칙 게시 순서로.

## 롤백

문제가 생기면 콘솔 규칙을 잠시 아래로 되돌리면 됩니다(임시 개방):

```json
{ "rules": { ".read": true, ".write": true } }
```
